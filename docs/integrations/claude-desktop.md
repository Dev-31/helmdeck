# Claude Desktop

> **Status:** 🟡 Documented, not yet verified end-to-end
> Connect snippet generated by `/api/v1/connect/claude-desktop` (`internal/api/connect.go`). Promote to ✅ once a maintainer has walked the Phase 5.5 loop.

## Prerequisites

Same as [claude-code.md](claude-code.md): running control plane, JWT, vault SSH key, fixture GitHub repo. Plus Claude Desktop installed: <https://claude.ai/download>.

## 1. Install the bridge

See [claude-code.md §1](claude-code.md#1-install-the-bridge) — same `helmdeck-mcp` binary.

## 2. Fetch the connect snippet

```bash
curl -s "http://localhost:3000/api/v1/connect/claude-desktop?token=$HELMDECK_TOKEN" | jq .
```

## 3. Configure Claude Desktop

Config path (per the snippet's `install_path`):

- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

Merge:

```json
{
  "mcpServers": {
    "helmdeck": {
      "command": "helmdeck-mcp",
      "env": {
        "HELMDECK_URL": "http://localhost:3000",
        "HELMDECK_TOKEN": "<your-jwt>"
      }
    }
  }
}
```

Quit and relaunch Claude Desktop (full quit, not just close window). The hammer icon should list the helmdeck packs.

## 4. Walk the Phase 5.5 code-edit loop

Identical to [claude-code.md §4](claude-code.md#4-walk-the-phase-55-code-edit-loop). Same prompt, same pass criteria.

## Troubleshooting

- **Hammer icon missing** — config JSON has a syntax error; Claude Desktop silently skips invalid configs. `jq . claude_desktop_config.json` to validate.
- **Bridge spawns but no tools listed** — check `~/Library/Logs/Claude/mcp*.log` for stderr from `helmdeck-mcp`.
- See [claude-code.md §Troubleshooting](claude-code.md#troubleshooting) for the rest.
