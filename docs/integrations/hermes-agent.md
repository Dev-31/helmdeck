# Hermes Agent

> **Status:** 🟡 Documented, not yet verified end-to-end
> Connect snippet generated by `/api/v1/connect/hermes-agent` (`internal/api/connect.go`). Hermes is the first client whose native config is YAML — the snippet is returned as JSON for transport, the example below shows the YAML form to paste. Promote to ✅ once a maintainer has walked the Phase 5.5 loop.

## Prerequisites

Same as [claude-code.md](claude-code.md): running control plane, JWT, vault SSH key, fixture GitHub repo. Plus Hermes Agent installed — see <https://github.com/NousResearch/hermes-agent>.

## 1. Install the bridge

See [claude-code.md §1](claude-code.md#1-install-the-bridge) — same `helmdeck-mcp` binary.

## 2. Fetch the connect snippet

```bash
curl -s "http://localhost:3000/api/v1/connect/hermes-agent?token=$HELMDECK_TOKEN" | jq .
```

The snippet's `format` field is `"yaml"` to flag that the JSON `config` block needs to be translated before pasting into Hermes' config file.

## 3. Configure Hermes Agent

Config path: `~/.hermes/config.yaml`. Hermes reads MCP servers from a top-level `mcp_servers:` map (per the [Hermes MCP feature docs](https://hermes-agent.nousresearch.com/docs/user-guide/features/mcp) and the [Use MCP with Hermes guide](https://hermes-agent.nousresearch.com/docs/guides/use-mcp-with-hermes/)). Merge:

```yaml
mcp_servers:
  helmdeck:
    command: helmdeck-mcp
    env:
      HELMDECK_URL: "http://localhost:3000"
      HELMDECK_TOKEN: "<your-jwt>"
```

Restart Hermes and ask the agent "What MCP tools do you have available?" or run `/tools` — you should see the helmdeck pack list.

## Troubleshooting

- **`/tools` does not list helmdeck packs** — Hermes silently skips entries with YAML errors. Validate with `yamllint ~/.hermes/config.yaml`.
- **Bridge spawns but every call returns `unauthorized`** — JWT expired or wrong scope; mint a new one in the API Tokens panel.
- See [claude-code.md §Troubleshooting](claude-code.md#troubleshooting) for the rest.
