## Prerequisites

- A running Helmdeck control plane (`make smoke` or a Compose stack from `deploy/compose/`).
- The control-plane URL (e.g. `http://localhost:3000`) and a JWT from the **API Tokens** panel.
- Claude Code installed: <https://claude.com/claude-code>.
- A private GitHub repo you can push to, plus an SSH deploy key loaded into the **Credential Vault** as type `ssh-git` (see ADR 022 + the Vault panel in the UI).

## 1. Install the bridge

Pick whichever channel matches your platform — all four ship the same `helmdeck-mcp` binary (T303 / T305–T307):

```bash
# Homebrew (macOS / Linux)
brew install tosin2013/helmdeck/helmdeck-mcp

# Scoop (Windows)
scoop bucket add helmdeck https://github.com/tosin2013/scoop-helmdeck
scoop install helmdeck-mcp

# npm
npm install -g @helmdeck/mcp-bridge

# Go
go install github.com/tosin2013/helmdeck/cmd/helmdeck-mcp@latest
```

Verify:

```bash
helmdeck-mcp --version
```

## 2. Fetch the connect snippet

```bash
curl -s "http://localhost:3000/api/v1/connect/claude-code?url=http://localhost:3000&token=$HELMDECK_TOKEN" | jq .
```

This returns the exact JSON to merge into Claude Code's MCP config.

## 3. Configure Claude Code

Claude Code reads MCP servers from `~/.config/claude/mcp.json` (the `install_path` field in the snippet). Merge the `config` block:

```json
{
  "mcpServers": {
    "helmdeck": {
      "command": "helmdeck-mcp",
      "env": {
        "HELMDECK_URL": "http://localhost:3000",
        "HELMDECK_TOKEN": "<your-jwt>"
      }
    }
  }
}
```

Restart Claude Code. Run `/mcp` and confirm `helmdeck` shows as connected with the full pack list (`browser.*`, `web.*`, `repo.*`, `fs.*`, `cmd.*`, `git.*`, `http.*`, …).

## 4. Walk the Phase 5.5 code-edit loop

The exit gate for milestone v0.5.5 is that an LLM can drive this loop end-to-end through the bridge, with the SSH key never appearing in the LLM's context window and every step audit-logged.

Prompt Claude Code with something like:

> Use the helmdeck packs to:
> 1. `repo.fetch` `git@github.com:<me>/<fixture-repo>.git` using vault credential `gh-deploy-key`.
> 2. `fs.list` the clone for `*.md` files.
> 3. `fs.read` the README and propose a one-line edit.
> 4. `fs.patch` to apply the edit (literal search-and-replace).
> 5. `cmd.run` `go test ./...` (or any project check) in the clone.
> 6. `git.commit` with message `chore: helmdeck integration smoke`.
> 7. `repo.push` back to `origin`.

**Pass criteria:**

- The new commit lands on the remote branch.
- `helmdeck audit tail` (or the **Audit Logs** panel) shows one entry per pack call, in order.
- Grep the Claude Code session transcript — the SSH private key must **not** appear anywhere; only the `${vault:gh-deploy-key}` placeholder.

If all three hold, update the status banner at the top of this file to ✅ with today's date + the Helmdeck version, and flip the matching row in [`README.md`](README.md).

## Troubleshooting

- **`helmdeck` server fails to start in `/mcp`** — run `helmdeck-mcp` directly in a shell with the same env vars; the bridge prints version-skew warnings (T304) and connection errors to stderr.
- **`unauthorized` on every pack call** — JWT expired or wrong scope. Mint a new one in the API Tokens panel.
- **`repo.fetch` returns `invalid_input: ssh key not found`** — the credential name in the prompt must match the vault entry exactly (case-sensitive).
- **`git.commit` returns `invalid_input: nothing to commit`** — `fs.patch` was a no-op; check the search string matches the file contents byte-for-byte (it's literal, not regex).
