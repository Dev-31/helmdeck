## 2. Fetch the connect snippet

```bash
curl -s "http://localhost:3000/api/v1/connect/openclaw?token=$HELMDECK_TOKEN" | jq .
```

OpenClaw uses TOML, but the connect endpoint returns the equivalent JSON shape — translate or use the snippet as a reference.

## 3. Configure OpenClaw

Config path: `~/.openclaw/openclaw.json`. OpenClaw nests MCP servers under each agent's `mcp.servers[]` array (it does **not** read a `~/.openclaw/mcp.toml`). Merge the helmdeck entry into your existing agent — typically the `main` agent:

```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "mcp": {
          "servers": [
            {
              "name": "helmdeck",
              "command": "helmdeck-mcp",
              "env": {
                "HELMDECK_URL": "http://localhost:3000",
                "HELMDECK_TOKEN": "<your-jwt>"
              }
            }
          ]
        }
      }
    ]
  }
}
```

Restart OpenClaw and confirm the helmdeck packs appear in the tool picker.

References:
- [OpenClaw MCP CLI docs](https://docs.openclaw.ai/cli/mcp)
- [How to Add MCP Servers on OpenClaw](https://openclawvps.io/blog/add-mcp-openclaw)
